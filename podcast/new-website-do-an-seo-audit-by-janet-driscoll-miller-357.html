New Website? Do an SEO Audit! by Janet Driscoll Miller #357

Ep. 35738 min2017-01-27

Before relaunching a website, run an SEO audit covering analytics, technical review, content, links, and benchmarking to prevent ranking drops and traffic loss after launch.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Copy all analytics tracking codes, including remarketing and AdWords conversion scripts, to the new site before launch.
  • Use Google Tag Manager during a relaunch to consolidate multiple analytics scripts into one manageable container.
  • A new CMS with too many plugins can slow page load enough to drop high-traffic keyword rankings to page two.
  • Set up 301 redirects using a tool like Xenu to map old URLs to new ones before going live.
  • Benchmark SEO metrics before relaunch so you can identify the cause of any traffic or ranking changes afterward.
a proper SEO audit identifies potential SEO pitfalls prior to launch, giving you time to get issues fixed before you replace your current website

About this episode

Redesigned your site? Moving to a new platform? Do an SEO audit with the help of Chris and Chuck, your SEO experts! They cover analytics, content review, links and more so you can arm yourself with a plan.

Questions this episode answers

What should an SEO audit cover before relaunching a website?

According to Janet Driscoll Miller's framework discussed on the podcast, a pre-launch SEO audit should at minimum cover analytics, technical review, content, links, and benchmarking. Each area helps prevent ranking loss or traffic gaps that can occur when a new site goes live.

How do you handle 301 redirects during a website relaunch?

The hosts recommend using a tool called Xenu to crawl both the old and new site, export all URLs into a spreadsheet, match old URLs to new ones, and then upload that spreadsheet to a redirect manager like Yoast. Broken or missing 301s can kill rankings and traffic.

Why is benchmarking important before a site relaunch?

Benchmarking records your SEO baseline before launch so you can monitor changes afterward. Without it, if traffic drops post-launch, you have no reference point to diagnose what caused the problem or to measure whether changes like a new PPC campaign are affecting organic results.

Should you wait to fix all content before relaunching a website?

The hosts say launching with old content on a cleaner, faster site already produces a ranking pop. They recommend launching first, then creating an editorial calendar to update pages over the following months, which can generate repeated ranking improvements as fresh content is published.

Terms defined in this episode
GTM (Google Tag Manager)
A tool that consolidates multiple analytics scripts, such as Google Analytics, remarketing, and AdWords conversion tracking, into one script managed through a single tag management interface.
301 redirect
A permanent redirect that sends users and search engines from an old URL to a new one, preserving link equity and preventing 404 errors during a site relaunch.
Shiko
An eWebResults branded term for social engagement, standing for shares, likes, and follows across social media platforms.
Benchmarking
Recording baseline SEO and traffic metrics before a site relaunch or campaign launch so that post-launch changes can be measured and diagnosed accurately.
