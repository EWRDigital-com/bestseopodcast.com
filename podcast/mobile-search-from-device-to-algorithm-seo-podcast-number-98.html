Key takeaways
  • Mobile search engines use completely separate bots and algorithms from traditional web search engines.
  • Creating a handheld.css stylesheet lets your existing site render on mobile without building separate content pages.
  • Mobile search engines place heavy weight on bounce rate because their algorithms are less refined than desktop ones.
  • If your site has large file sizes, sloppy code, or lots of multimedia, a separate mobile site is needed over a stylesheet.
  • Avoid mobile platforms that host your site under their own domain, as all link juice accumulates to their domain, not yours.

Questions this episode answers

What is a handheld.css file and how does it help with mobile SEO?

A handheld.css stylesheet formats your existing website pages for mobile viewing without creating separate mobile content. It lets you keep the SEO value already built into your site. Most mobile phones except iPhones will automatically apply it, though it does not work every single time.

When should you build a completely separate mobile website instead of using a stylesheet?

According to the episode, if your site has large file sizes, sloppy code, or lots of multimedia content, a separate mobile site is needed. Sites relying heavily on JavaScript, Flash, or Ajax will likely not load properly even with a handheld stylesheet in place.

What is the risk of using third-party software to mobilize your website?

Some web-based mobile platforms host your mobile site under their own domain, so all link juice and rankings accumulate to their domain rather than yours. You also risk bad file names being generated, which further strips away SEO value from your mobile pages.

How do mobile search engine algorithms differ from traditional search engine algorithms?

Mobile search engines use entirely separate bots and algorithms. They are less refined than traditional engines and place heavier weight on bounce rate as a ranking signal. Because of this, following traditional SEO best practices like title tags, header tags, and alt tags still applies to mobile pages.

Terms defined in this episode
handheld.css
A mobile style sheet that formats existing website pages for viewing on a mobile phone without requiring separate mobile content, while preserving the site's existing SEO value.
mobile sitemap
A sitemap specifically for mobile content that should be linked into your robots.txt file, helping mobile search engine bots crawl and index your mobile pages.
mobile subdomain
A separate subdomain such as mobile.ewebstyle.com used to host a mobile version of a site, which can help avoid duplicate content issues flagged by mobile search engines.
mobile bounce rate
A signal mobile search engines weigh heavily in rankings because their algorithms are less refined, making it important to design mobile sites that keep users engaged.
