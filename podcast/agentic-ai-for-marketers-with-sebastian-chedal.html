Agentic AI For Marketers With Sebastian Chedal

Episode55 min2026-04-19Guest: Sebastian Chedal

Agentic AI systems for marketers work best when built with clear job descriptions, state-driven memory, scripted skills, and tiered risk approval, compressing the gap between idea and delivery.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Define a job description with input, process, and output format before choosing any AI architecture or tier level.
  • Script repetitive agent actions so AI only handles interpretation, cutting errors and token costs.
  • Use tiered risk approval: auto-approve low-risk actions, checkpoint medium-risk at the end, gate high-risk at start and end.
  • Content scored as commodity, meaning it would not be missed if removed from the web, should be blocked from publishing pipelines.
  • Spend roughly double the historical planning time upfront, writing product or architecture design documents before agentic coding begins.
The compression really is between idea and delivery. And another way to look at it would be problem solution or opportunity and action.

Questions this episode answers

What are the main tiers of AI agent usage for marketers?

Tier one is AI assistance for individual tasks. Tier two is AI inside workflow automations triggered on a schedule. Tier three is fully agentic AI with memory, tools, and the ability to take its own actions, either driving a computer locally or running always-on in the cloud on servers like AWS EC2.

How should marketers handle persistent memory in AI agent systems?

Sebastian recommends state-driven storage such as a SQL database, JSON files, or a relational table rather than letting the AI track state in a markdown file. Memory must be managed carefully because context windows are limited, and too many skills reduce available input tokens for each task.

Why do AI agents sometimes ignore explicit instructions or hard-code values?

Possible causes include model mismatch where a weaker model like Haiku relies on pattern replication, poor context or skill management, and prompts that specify steps without explaining the goal. Stronger models like Opus follow instructions more reliably, and communicating the purpose of a task reduces shortcuts like hard-coded magic numbers.

What are the four human roles that remain valuable as agentic AI expands?

Sebastian identifies direction or taste, interaction with clients and stakeholders, building the underlying agent infrastructure, and supporting and maintaining that infrastructure. Direction and interaction are horizontal roles cut across all disciplines, while building and supporting are vertical roles focused on the specialized systems that create competitive advantage.

Terms defined in this episode
Agentic AI
An AI system that has memory, tools, and agency, meaning it can take its own actions either on a schedule or when triggered, without requiring a human to initiate each step.
Context firewalling
Keeping the context window of one agent separate from another to prevent memory bleed and maintain controlled, predictable behavior across a multi-agent system.
Commodity content
Content that, if removed from the web, no one would notice or miss, meaning it has no novel research, unique framing, or authoritative value.
Token optimization
Managing the number of tokens consumed during agentic AI runs, for example by using commands like slash context, to control costs when agents perform many sequential actions.
