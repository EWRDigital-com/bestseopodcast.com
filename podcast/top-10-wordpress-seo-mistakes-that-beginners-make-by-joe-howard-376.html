Top 10 WordPress SEO Mistakes That Beginners Make by Joe Howard #376

Ep. 37636 min2017-06-06

WordPress beginners should avoid 10 common SEO mistakes including skipping XML sitemaps, ignoring schema markup, neglecting open graph metadata, and failing to set permalinks to post names for better rankings.

Key takeaways
  • Set your WordPress time zone in general settings so scheduled posts publish with the correct date and time.
  • Submit your XML sitemap to both Google Search Console and Bing Webmasters after every new site build.
  • Use open graph social metadata so Facebook and other platforms display the correct image and description when content is shared.
  • Set goals in Google Analytics based on actionable items like contact form submissions or downloads, not vanity metrics like time on site.
  • Implement schema markup at minimum for name, address, and phone, and expand to events or menus depending on your business type.
Regularly means whatever you can commit to that will allow you to produce quality content.

Questions this episode answers

What is the most important permalink setting in WordPress for SEO?

You should set permalinks to the post name, giving you a clean URL like yoursite.com slash post title. Avoid including categories, dates, or page IDs in the URL because they create long, confusing links that hurt both user experience and search rankings.

Why should WordPress site owners add open graph social metadata?

Open graph metadata tells social platforms like Facebook exactly which image and description to display when a page is shared. Without it, Facebook may pull a random image or truncate content unpredictably, reducing the click-through appeal of your shared posts.

What kinds of goals should you set in Google Analytics for a WordPress site?

Set goals based on actionable items such as contact form submissions, downloads, or phone calls. Avoid setting goals for vanity metrics like average time on site, because those can be inflated by users who open a page and get distracted without meaningfully engaging.

What is schema markup and why does it matter for WordPress SEO?

Schema markup is structured data that influences how your pages appear in search results. The majority of websites do not use it at all. At minimum you should mark up your name, address, and phone number, and expand to menus, events, or locations depending on your business type.

Terms defined in this episode
Open Graph Social Metadata
Data embedded in a webpage that tells social platforms like Facebook which image, title, and description to display when the page is shared, ensuring the correct content appears rather than a random selection.
XML Sitemap
A file described as a big list of every post, page, and piece of media on your website that you submit to Google through Search Console to tell them everything you want indexed.
Schema Markup
Structured data added to a webpage that influences how the page shows up in search results, covering details like name, address, phone, events, menus, and locations.
Permalinks
The permanent URLs for WordPress posts and pages. Setting them to the post name format produces clean, keyword-friendly URLs that are easier for Google to index and for users to understand.
