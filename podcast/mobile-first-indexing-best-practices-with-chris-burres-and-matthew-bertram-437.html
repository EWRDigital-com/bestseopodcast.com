Questions this episode answers

What is mobile-first indexing and how does it affect my website rankings?

Mobile-first indexing means Google now crawls and ranks your site based on the mobile version rather than the desktop version. If your mobile site has less content, missing metadata, or no structured data compared to your desktop, your rankings will drop when Google switches your site over.

Do I need two separate indexes for desktop and mobile?

No. Google continues to use only one index. It has switched from indexing the desktop version to indexing the mobile version. Sites with separate desktop and mobile URLs, such as m-dot subdomains, are most at risk if the two versions differ in content.

What should I check on my site to prepare for mobile-first indexing?

Check that your mobile site has the same primary content, images, alt attributes, videos, structured data, and metadata as your desktop version. Also verify both site versions in Search Console, review canonical tags, hreflang links, robots.txt files, and sitemaps for both versions.

What is the data highlighter in Google Search Console and why does it matter for mobile-first indexing?

The data highlighter is a Search Console tool that lets you tag pieces of data on your pages, such as event dates, times, costs, and locations, so Google understands that information as structured data. It is an alternative to adding schema code directly and should be applied to both mobile and desktop versions.

Terms defined in this episode
Mobile-first indexing
Google's approach of crawling and indexing pages using a smartphone agent rather than a desktop agent, meaning the mobile version of a site determines search rankings.
Canonical tag
An HTML element that points to the originator of content. With mobile-first indexing, desktop pages should use a canonical tag pointing to the mobile version so the mobile URL is treated as the original source.
Data highlighter
A Google Search Console tool that lets site owners tag pieces of page content, such as event dates, times, costs, and locations, to communicate structured data to Google without writing schema code.
Responsive web design
A design approach where page content adjusts based on the size of the device being used, covering mobile, tablet, and desktop screen sizes within a single URL.
