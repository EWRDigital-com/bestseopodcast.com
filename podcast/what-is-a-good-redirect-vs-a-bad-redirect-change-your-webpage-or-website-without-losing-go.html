Questions this episode answers

What is a 301 redirect and when should I use it?

A 301 redirect is a server-level redirect that transfers Google ranking credit from an old URL to a new one. Use it whenever you move a page to a new location, change a domain name, or reorganize internal site folders, so existing link value is not lost.

What makes a redirect bad for SEO?

An HTTP meta-refresh redirect placed in a default HTML page is a bad redirect because Google penalizes it. The server delivers the default page and that page tells the browser to load a different URL, which search engines flag as a manipulative technique.

How did Kay Bailey Hutchinson's website get banned from Google?

The site used CSS to hide a large block of text from visitors while keeping it visible to search engines, a technique called hidden text combined with keyword stuffing. Google specifically banned the site for those black-hat techniques, including irrelevant terms like Texas BBQ Pit and chicken fried steak recipe.

How do you fix a homepage that incorrectly redirects to an inner page?

Move the homepage content into the default or index HTML file so the root directory loads it directly. Then set up a 301 redirect from the old inner page to the new default page, preserving any link credit the inner page had already accumulated.

Terms defined in this episode
301 redirect
A server-level redirect that tells search engines a page has permanently moved and transfers ranking credit from the old URL to the new one.
HTTP meta-refresh redirect
A redirect placed inside an HTML page that instructs the browser to load a different URL, considered a bad redirect that Google penalizes.
hidden text
A black-hat technique where CSS or other styling places images over large blocks of text so visitors cannot see the text but search engines can index it.
robots.txt
A text file placed on a server that tells search engines where to go, where not to go, and how often to crawl, with no purpose other than controlling search engine behavior.
