Questions this episode answers

What are the Core Web Vitals metrics Google uses for page experience?

The three Core Web Vitals are Largest Contentful Paint (LCP), which measures visual loading speed, First Input Delay, which measures interactivity, and Cumulative Layout Shift (CLS), which measures visual stability as a page loads. Other page experience signals include mobile friendliness, safe browsing, HTTPS, and non-intrusive interstitials.

How do I find long tasks hurting my page speed in Chrome DevTools?

Right-click the page, select Inspect, go to the Performance tab, and click Record. Long tasks appear at the top of the waterfall chart under the Main tab and are highlighted with a red triangle. Click a long task and open the Bottom Up tab to see which specific script or resource is responsible.

What causes cumulative layout shift and how do I fix it?

CLS is caused by page elements moving around as the page loads, such as a cookie consent banner pushing content down because no space was reserved for it. To fix it, reserve space in your layout for images, embeds, and banners so they do not shift existing content when they appear.

How should I evaluate whether a WordPress plugin is safe to use?

Look for at least 10,000 downloads to confirm active usage, and verify the developer has posted an update or responded to a user question within the last three months. Plugins that go inactive stop receiving security patches, leaving holes that hackers can exploit.

Terms defined in this episode
Largest Contentful Paint (LCP)
A Core Web Vitals metric that measures how quickly the main on-page elements load, representing the first indicator for a user that a page is loading.
Cumulative Layout Shift (CLS)
A Core Web Vitals metric that looks at the amount the visual layout of a page moves around as the page loads, measuring user frustration from unexpected shifts.
Interstitial
A banner, pop-up, or overlay that loads on top of or pushes down page content, obstructing crucial on-page content for users as they browse.
Long Task
A piece of JavaScript code that blocks the main browser thread for a long period of time, visible as a red triangle in the Chrome DevTools performance waterfall chart.
