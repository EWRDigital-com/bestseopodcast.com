Key takeaways
  • Submit local events to high-PR outlets like newspapers to earn followed links that stay live as archived pages.
  • Use schema.org microdata tags to clearly identify product or event attributes so Google can surface them in relevant searches.
  • For e-commerce sites, prioritize Google Base to get products into Google Shopping over investing time in microdata markup.
  • Diversify anchor text in link profiles so that not every inbound link uses the same keyword-rich anchor text.
  • Panda 3.9 released July 24th targets sites with irrelevant or over-optimized content, so freshening and rewriting content reduces penalty risk.

Questions this episode answers

What is microdata and should I use it on my e-commerce site?

Microdata is snippets of HTML that let you tag specific attributes of content, such as product title, category, and description, so Google can clearly identify what the page is about. For e-commerce and events it can improve search visibility, but the hosts suggest prioritizing Google Base for product listings first.

What is LSI and how does it affect keyword optimization?

LSI stands for latent semantic indexing. Google uses it to associate related terms, so searching for one word can surface results for closely related words. It affects long-tail phrases and misspellings, and the context of other words in a search query helps Google decide which meaning and results are most relevant.

How can local events help with link building?

Submitting events to local newspaper websites and directories earns links from high-PR, well-branded domains. Those event listing pages typically become archived after the event, so the link persists. Listings on sites like Craigslist also provide local citations with name, address, phone number, and a link back to your site.

What did Google's Panda 3.9 update target?

Panda 3.9 was released July 24th and follows updates 3.8 and 3.7 released in June. The update targets sites with irrelevant or over-optimized content. The hosts recommend rewriting and freshening content to make it more relevant and less spammy to avoid being affected.

Terms defined in this episode
Microdata
Snippets of HTML used to tag specific attributes of content, such as event date or product category, allowing Google to clearly identify and display that content in search results.
LSI (Latent Semantic Indexing)
A database-style algorithm component Google uses to associate related terms and context clues across indexed pages, influencing which results appear for a given search query.
Schema.org
A website providing a hierarchy of microdata tags that webmasters can use to mark up content attributes like event name, date, time, and location for better search visibility.
Diverse Link Profile
A backlink portfolio that mixes anchor text types, including keyword-rich anchors, generic phrases like click here, and brand names, so the link pattern appears natural to search engines.
