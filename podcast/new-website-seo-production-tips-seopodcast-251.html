New Website SEO Production Tips - #SEOpodcast 251

Episode34 min2014-10-22

Building a new website for SEO requires simple code, integrated blogging on the same domain, fast load times, responsive design, clear navigation, and a marketing plan before launch.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Clean WordPress installs, minified CSS and JavaScript, and removing unused plugins reduce crawl errors and speed up sites.
  • Blog content should live on the same domain name, not a separate domain, to consolidate SEO value.
  • Site speed should be three seconds or less on desktop and six seconds or less on mobile to prevent bounce rate spikes.
  • Responsive design eliminates duplicate content issues, but default bootstrap layouts may hurt mobile conversions without custom styling.
  • Plan your marketing funnel before building the site so landing pages, navigation, and content match how customers actually convert.

Questions this episode answers

How fast should a new website load for SEO?

According to this episode, desktop sites should load in about three seconds or less, and mobile sites should load in six seconds or less. Slower load times cause bounces to increase exponentially and create a negative user experience that prevents conversions even when visitors do stay on the page.

Should a blog be on the same domain or a separate domain?

The episode says that in 2014 it is worth having all blog content on the same domain name rather than a separate domain. Having the blog on the same domain keeps the same look, feel, and voice, and consolidates the SEO value of that content to one site.

What is the difference between retargeting and remarketing?

According to the episode, retargeting and remarketing are the same thing. The only difference is that remarketing is Google's term for it, used inside AdWords. Facebook, Bing, and Twitter all call it retargeting. If someone says remarketing, they are most likely referring to Google AdWords specifically.

What are the biggest SEO mistakes to avoid when building a new website?

This episode highlights excess or unclean code, hosting a blog on a separate domain, ignoring site speed, using default responsive layouts without testing mobile usability, poor navigation including drop-down menus that break on touch devices, and failing to plan the marketing strategy before the site is built.

Terms defined in this episode
Retargeting
Showing ads to people who have already shown interest, such as by visiting your site or searching related keywords. Facebook, Bing, and Twitter use this term. Google calls the same process remarketing, which is only used inside AdWords.
Responsive Design
A website design approach that adapts layout across screen sizes. The episode notes it eliminates duplicate content issues but warns that default bootstrap responsive setups may not work correctly for mobile conversions without custom styling.
Breadcrumbs
Navigation elements that show users the path they took through a site and allow them to backtrack steps, especially useful on large sites with many categories and pages.
WYSIWYG Editor
A what you see is what you get editor, such as the WordPress visual tab, that can add unnecessary inline code and span tags instead of clean styled tags, hurting code simplicity.
