Choose a CMS like Wordpress for SEO - SEO Podcast - Number 81

Episode33 min2010-11-23

When choosing a CMS for SEO, prioritize systems that support clean keyword-rich URLs, H1 tags, customizable URL structures, and 301 redirects, with WordPress and Joomla both capable but WordPress easier to configure.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Clean, keyword-rich URLs matter for both SEO rankings and reading analytics reports clearly.
  • Your CMS must allow H1 tag insertion on every page, as search bots pay special attention to them.
  • 301 redirects consolidate link juice from duplicate URLs like www and non-www versions of a homepage.
  • WordPress is easier to find and configure SEO features than Joomla, though Joomla supports them too.
  • Dates in URL structures may cause visitors to skip older but still valuable content.
You want to be able to customize that with keywords into a keyword-rich URL.

Questions this episode answers

What CMS features are most important for SEO?

According to this episode, the four critical CMS features for SEO are: URLs free of tracking parameters and session IDs, the ability to insert H1 tags, a customizable keyword-rich URL structure, and support for 301 redirects to a canonical URL to consolidate link juice.

Is WordPress good for SEO?

Yes. The episode says WordPress supports all key SEO features through plug-ins, including friendly URLs and optimum title controls. The page title in WordPress also automatically becomes the H1 tag. The main complaint is frequent updates that can break existing plug-ins.

What is a 301 redirect and why does it matter for SEO?

A 301 redirect tells Google to send all link juice from one URL to another. The episode uses the example of ewebstyle.com and ewebstyle.com/index.asp being treated as two separate pages, splitting link value. A 301 redirect consolidates that value to one preferred URL.

Should you include dates in your CMS URL structure?

The episode says to think carefully before including dates. Dates can cause visitors to skip older articles assuming the content is outdated, even when it is still valuable. The hosts were in the process of removing dates from their own blog URLs when this episode was recorded.

Terms defined in this episode
Canonical URL
The preferred version of a URL that should receive all link juice, used with 301 redirects to prevent the same page from being indexed under multiple addresses.
301 Redirect
A server instruction that permanently points one URL to another, consolidating link value so Google juice is not split across duplicate pages.
Permalink
A permanent URL setting in WordPress that replaces default ugly URLs like article-equals-two with keyword-rich, human-readable addresses.
Session ID
A tracking parameter appended to a URL that makes it look ugly, prevents keyword inclusion, and can cause search engines to treat the same page as multiple separate pages.
