#SEOpodcast 273 - Common SEO Mistakes You Can Avoid

Episode31 min2015-05-06

When redesigning a website, avoiding five common SEO mistakes including unblocked dev sites, broken analytics, URL changes, conflicting plugins, and misaligned keyword strategy can prevent steep drops in organic traffic.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Block development and staging sites from indexing using a robots.txt file to prevent duplicate content issues.
  • After a redesign, verify analytics is installed correctly and update goals to match any new URL structures.
  • 301 redirect all old URLs to their new equivalents so link equity is preserved after a redesign.
  • Avoid activating multiple SEO plugins simultaneously since they conflict over which title tags to display.
  • If messaging changes during a redesign, rewrite, re-optimize, and redirect to keep content and keyword strategy consistent.

Questions this episode answers

How do you prevent a development site from hurting SEO during a redesign?

Use a robots.txt file to block all development or staging URLs from being crawled, or add a noindex meta tag to each page. Remove those blocks at launch. This prevents duplicate content and broken pages from appearing in Google search results.

What should you do when your site messaging changes during a redesign?

Rewrite the content to match the new messaging, then re-optimize all titles, meta information, images, and internal links to communicate the same message. Finally, redirect URLs as needed so the old optimization and the new messaging do not conflict for users or search engines.

Terms defined in this episode
301 redirect
A permanent redirect that points an old URL to a new one, preserving link equity and preventing broken links after a site redesign.
robots.txt
A text file that tells search engines which pages or sections of a site to block from crawling and indexing, used here to prevent development sites from being indexed.
noindex meta tag
A meta tag placed on individual pages that explicitly tells search engines not to index that page, used as an alternative to robots.txt for blocking development content.
link equity
The value passed through inbound links pointing to specific pages, which can be lost if URLs change without proper 301 redirects during a site redesign.
