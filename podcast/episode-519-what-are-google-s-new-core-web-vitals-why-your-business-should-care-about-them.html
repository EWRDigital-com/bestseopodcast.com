Questions this episode answers

What are Google's Core Web Vitals and what do they measure?

Core Web Vitals are three metrics Google uses to measure user experience: Largest Contentful Paint measures page loading time, First Input Delay measures how quickly the browser responds to user interaction, and Cumulative Layout Shift measures visual stability while content loads.

How can I fix a poor Cumulative Layout Shift score?

Specify width and height dimensions for images and video elements, reserve space for dynamic ad units by reviewing historical ad sizes, use placeholder elements for dynamic content like banners, and preload web fonts so they do not cause content to shift when they load.

Where can I check my website's Core Web Vitals data?

Core Web Vitals are available in Google Search Console, where individual URLs are shown in green, yellow, or red for both mobile and desktop. The data is also accessible through PageSpeed Insights and Google developer tools, including both field data and lab data.

Terms defined in this episode
First Input Delay (FID)
A Core Web Vital that measures the delay between when a user first clicks on a site and when the browser is able to respond to that interaction. Under 100 milliseconds is considered good.
Cumulative Layout Shift (CLS)
A Core Web Vital measuring visual stability. It is calculated as the impact fraction times the distance fraction. A score below 0.1 is good and above 0.25 is poor.
Field Data
Real-world performance data collected from actual Chrome browser users as pages load, which can vary by device and connection speed, as opposed to controlled lab data.
