Key takeaways
  • Daisy-chained redirects can slow a site dramatically. Fixing three redirect chains took one site from eight or nine seconds to under three.
  • Unoptimized images are a top culprit. Mobile phone photos uploaded directly to WordPress are massive and must be compressed before use.
  • Lazy loading above-the-fold content does not negatively impact page load metrics and can improve user experience by showing progress.
  • Enterprise SEO requires building a culture of SEO across departments, getting cross-functional approvals, and possibly adding a developer to the marketing team.

Terms defined in this episode
Browser Caching
Storing static assets like logos, styles, and images on a user's hard drive for a set period so repeat pages do not re-download those resources, leading to faster load times.
Content Delivery Network (CDN)
A group of geographically distributed servers that work together to deliver site content quickly, reducing load times, bandwidth consumption, and in some cases adding a layer of security.
Lazy Loading
A technique that prioritizes loading content the user will see or interact with first, rather than loading all page content simultaneously, improving perceived speed without hurting load metrics.
Persistent Opcode Cache
A server-side cache for PHP applications that stores human-readable code so it does not have to be recompiled on every request, speeding up PHP-based sites like WordPress.
