Dangers of dynamic content with Google. Should I use frames? - Unknown Secrets of SEO E-Webstyle Number 34

Episode33 min2009-10-31

Dynamic content URLs with mod_rewrite and frames both confuse Google bots, so use SEO-friendly URL structures, text link pyramids, and avoid frames entirely to protect crawlability and keyword relevance.

Key takeaways
  • Multiple domains forwarding to one site create duplicate content and will not both show in Google results simultaneously.
  • Use mod_rewrite to create slash-separated SEO-friendly URLs that include keywords instead of stripping all query parameters.
  • Frames are collections of independent websites that confuse search engines, so avoid them entirely.
  • A text link pyramid at the bottom of every page passes keyword-relevant Google juice that a sitemap alone cannot provide.

Questions this episode answers

Is it bad for SEO to have multiple domain names forwarding to one website?

Google treats the forwarded domains as duplicate content and will not show both in results at the same time. The benefit is that a keyword-rich domain name like ImproveMyCreditDallas.com can still rank higher for that specific term than the main domain, so it is not a penalty but offers limited value without unique content on each domain.

Does using mod_rewrite to clean up PHP or ASP URLs hurt Google crawling?

If mod_rewrite collapses all dynamic query-string URLs into one generic page name like slash testing, Google assumes it already indexed that page and ignores new content served there. The recommended approach is to rewrite URLs into keyword-rich slash-separated paths so each unique page has a distinct, crawlable URL.

What is a link pyramid and why does it help SEO?

A link pyramid is a set of plain text links placed at the bottom of a webpage that mirrors the site navigation. It ensures Google can follow links even when navigation uses images or JavaScript it cannot parse, and it passes keyword-relevant authority from the page to its internal link destinations.

Terms defined in this episode
mod_rewrite
Server-side software that rewrites dynamic URLs, typically used to convert query-string addresses into cleaner, keyword-rich slash-separated URLs that search engines can index more effectively.
Supplemental Index
A secondary Google index where pages with duplicate or low-value content are stored, making them unlikely to appear in standard search results.
Link Pyramid
A block of plain text navigation links placed at the bottom of a webpage so search engine bots can follow and assign keyword relevance to internal pages even when main navigation uses images or JavaScript.
Frames
An HTML technique that loads multiple independent web pages inside one browser window, divided by visible lines or scroll bars, which confuses search engine crawlers because each frame is a separate document.
