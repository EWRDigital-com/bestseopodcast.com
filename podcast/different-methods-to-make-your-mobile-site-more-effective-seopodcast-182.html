Different Methods to Make Your Mobile Site More Effective - #seopodcast 182

Episode32 min2013-03-13

Mobile SEO works best when sites are built for mobile users from the start, prioritizing social sharing, clickable addresses, large navigation, and a subdomain structure so search engines do not flag mobile pages as duplicate content.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Develop mobile sites to scale for mobile users rather than retrofitting a desktop site into a smaller layout.
  • Make phone numbers, addresses, and maps clickable on mobile sites so users can act immediately.
  • Place mobile site content on a subdomain such as m.yourdomain.com so Googlebot recognizes it as mobile, not duplicate content.
  • Think social first on mobile by making it easy to share content to Facebook, Twitter, and other platforms.
  • Decide between an app and a mobile site based on whether you push frequent updates, events, or need deep social integration.
Search engines understand that mobile-friendly versions of your web pages are just that and not duplicate content.

Questions this episode answers

Should I build a separate mobile site or an app for my business?

According to this episode, it depends on your goals. If you have frequent events, products, and heavy social integration needs, an app may be the right direction. If you just have a lot of mobile traffic but no ongoing push content, a well-optimized mobile version of your site is likely sufficient.

How do I prevent my mobile site from being treated as duplicate content by Google?

Host Charles Lewis explains that you should put your mobile site on a subdomain, such as m.yourdomain.com, and declare it in your robots file as a mobile user agent. This tells Googlebot the content is a mobile version, not duplicate content, and lets you preserve link equity.

What are the most important features to include on a mobile website?

The episode highlights clickable phone numbers, a clickable address that links to a map, large and easy-to-tap navigation, a visible link to the desktop version, and prominent social sharing options. For brick-and-mortar businesses, contact and location features are described as key for the majority of mobile visitors.

How should images and rich media be handled on a mobile site?

The hosts say rich media such as video and images can help sell products and are acceptable on modern mobile devices. However, images should be properly optimized, meaning you should not upload large raw files directly from a camera. Heavily compressed, well-optimized files are recommended.

Terms defined in this episode
Mobile subdomain
Placing a mobile version of a site at a subdomain such as m.yourdomain.com so Googlebot recognizes it as a mobile site and does not treat its content as duplicate content from the main site.
Robots file user agent mobile
A robots.txt directive that identifies mobile Googlebot so the search engine knows to index that content as a mobile version rather than as a duplicate of desktop pages.
Click to call
A mobile site feature that lets a visitor tap a phone number to immediately dial the business, described in the episode as a key function that must be present on mobile sites.
Short-tail mobile queries
Brief search phrases that mobile users tend to type or speak because they are often multitasking or driving, making concise keyword optimization important for mobile SEO.
