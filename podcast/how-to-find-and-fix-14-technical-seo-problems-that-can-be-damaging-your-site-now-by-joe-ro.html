Key takeaways
  • Check Google Search Console Index Status to confirm your pages are actually indexed and catch unexpected drops or spikes.
  • A misplaced disallow in robots.txt or a meta robots no-index tag can block or remove pages from Google's index entirely.
  • HTTPS is becoming mandatory: Google marks non-secure sites collecting any user input as non-secure, and ranking benefits are growing.
  • 301 redirects remain the gold standard for permanently moved pages; avoid meta refresh redirects, which are an outdated tactic.
  • Internal links from menus, content, and footers to key pages help search spiders find and crawl your most important content.

Questions this episode answers

What is the difference between a robots.txt disallow and a meta robots no-index tag?

A misconfigured robots.txt prevents new pages from being indexed but will not remove pages already in Google's index. A meta robots no-index directive actively removes all pages carrying that tag from the index, making it a more damaging mistake if applied unintentionally.

Why does URL canonicalization matter for SEO?

Search engines treat different URL versions of the same page, such as www versus non-www or URLs with and without trailing slashes, as separate pages. This creates duplicate content problems. Picking one version and setting it as the default in Search Console avoids those issues.

How does hidden text cause technical SEO problems?

Text hidden with CSS such as display none is still read by Google bots. A page that appears to have a few hundred words but contains thousands of hidden words can trigger penalty concerns for possible cloaking, even if the intent was accidental rather than manipulative.

Terms defined in this episode
Indexation
Whether a site's pages have been included in Google's index and can appear in search results, verified through Google Search Console under Google Index Status.
rel=canonical
An HTML link tag that tells search engines which URL is the preferred version of a page, used to prevent duplicate content issues common on e-commerce sites with filtered URLs.
XML Sitemap
A file that helps Google and other search engine spiders understand and crawl a site's structure; must be correctly configured and submitted in Search Console and Bing Webmaster Tools.
Meta Refresh
A client-side redirect that automatically sends users to another page without a server-side redirect; considered an outdated tactic not recommended by Google or professional SEOs.
