Everything You Need to Know About WordPress and GDPR Compliance #423

Ep. 42326 min2018-06-11

GDPR applies to any website with EU visitors, requiring explicit consent for data collection, a right to data deletion, breach notifications within 72 hours, and cookie notices for tracking pixels.

Apple PodcastsSpotifyYouTubeiHeartRadioRSS
Key takeaways
  • Add a required consent checkbox to contact forms explaining exactly how submitted data will be used or stored.
  • For Google Analytics, either anonymize data before storage or add a cookie consent overlay for EU visitors.
  • Businesses must report certain data breaches to authorities within 72 hours under GDPR.
  • The maximum fine is $26.8 million or 4% of global profits, but enforcement starts with a warning before escalating.
Simply adding a required consent checkbox with clear explanation should be good enough for you to make your WordPress forms GDPR compliant.

Terms defined in this episode
GDPR
General Data Protection Regulation, a European Union law covering personal data such as name, email, IP address, health information, and income information, with fines up to $26.8 million or 4% of global profits.
Explicit Consent
A clear, unambiguous agreement from the user stating what will be done with their data, required before collecting or using personal information under GDPR.
Right to Be Forgotten
A GDPR requirement that businesses must be able to delete all stored data about a user if that user requests it.
Breach Notification
A GDPR obligation requiring organizations to report certain data breaches to authorities within 72 hours, and to inform affected individuals immediately if the breach is high risk.
